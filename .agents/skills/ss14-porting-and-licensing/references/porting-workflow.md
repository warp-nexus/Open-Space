# Porting Workflow

## Before Porting

- verify the source repository and license
- check whether the feature is stable enough to port
- start from an up-to-date branch

## Preferred Import Path

- prefer porting a reviewed PR or known commit range
- keep imported changes narrow
- adapt the feature to local folder structure and `_OpenSpace` placement instead of dumping foreign paths blindly

## PR Expectations

- say what was ported
- say from where
- say what was adapted
- say what was verified
